# Code of Conduct

`openyak` is a public engineering repository. Collaboration here should stay technically rigorous, direct, and respectful.

## Expected behavior

- discuss the code, tests, design, and evidence rather than attacking people
- keep reports grounded in reproducible facts
- be clear about assumptions, constraints, and tradeoffs
- accept correction when verification contradicts a claim
- respect the project's local-first and evidence-driven boundaries

## Unacceptable behavior

- harassment, intimidation, threats, or sustained hostility
- personal attacks, slurs, or demeaning language
- publishing someone else's private information without consent
- spamming, derailing, or repeatedly ignoring repository scope
- posting secrets, exploit payloads, or other sensitive material in public threads

## Moderation

Maintainers may edit, hide, lock, or remove content that undermines productive collaboration, and may restrict participation for repeated or serious violations.

## Reporting conduct issues

- For non-sensitive conduct issues, open a GitHub issue with enough context for maintainers to review the situation.
- For issues that involve private personal information or sensitive context, use the most private GitHub contact/reporting path available to this repository or the repository owner's public contact path if one is listed.

## Project-specific note

This repository values evidence over heat. Disagreement is expected; contempt is not.
